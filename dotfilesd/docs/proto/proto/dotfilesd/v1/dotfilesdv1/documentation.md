## Services

### dotfilesd.v1.DocumentationService

DocumentationService provides human-readable documentation for plugin RPC
services. The daemon CLI and MCP tools use this to generate help text,
usage hints, and parameter descriptions dynamically.

Plugins get a default implementation via the SDK that returns "not
implemented". They can override it to provide rich, per-service docs.

#### GetDocumentation

GetDocumentation returns documentation for a specific service.

- **Request:** `dotfilesd.v1.DocumentationRequest`
- **Response:** `dotfilesd.v1.DocumentationResponse`


## Messages

### DocumentationRequest

| Field | Type | Description |
|-------|------|-------------|
| `service_name` | string | Fully-qualified service name (e.g. "weather.WeatherService"). If empty, returns documentation for ALL services exposed by this plugin. |

### DocumentationResponse

| Field | Type | Description |
|-------|------|-------------|
| `format` | string | Documentation format. Defaults to "markdown". Other formats may be supported in the future ("html", "man"). |
| `content` | string | Documentation content in the requested format. |
| `documentation` | dotfilesd.v1.Documentation | Structured documentation, populated when the plugin has embedded docs from protoc-gen-docs. Preferred over format/content for programmatic use. |

### Documentation

Documentation is the structured representation of all documentation for a
proto package, generated by protoc-gen-docs and embedded in plugins.

| Field | Type | Description |
|-------|------|-------------|
| `package` | string | Proto package name (e.g. "weather"). |
| `description` | string | File-level leading comment from the proto file. |
| `services` | repeated dotfilesd.v1.ServiceDoc | Per-service documentation. |
| `messages` | repeated dotfilesd.v1.MessageDoc | Top-level message documentation (not nested under services). |
| `enums` | repeated dotfilesd.v1.EnumDoc | Top-level enum documentation. |

### ServiceDoc

ServiceDoc documents a single protobuf service.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Fully-qualified service name (e.g. "weather.WeatherService"). |
| `description` | string | Service description from proto leading comments. |
| `methods` | repeated dotfilesd.v1.MethodDoc | Per-method documentation. |

### MethodDoc

MethodDoc documents a single RPC method with full request/response docs.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Method name (e.g. "Forecast"). |
| `description` | string | Method description from proto leading comments. |
| `request_type` | string | Fully-qualified request type name (e.g. "weather.ForecastRequest"). |
| `response_type` | string | Fully-qualified response type name (e.g. "weather.ForecastResponse"). |
| `request` | dotfilesd.v1.MessageDoc | Full request message documentation with fields. |
| `response` | dotfilesd.v1.MessageDoc | Full response message documentation with fields. |

### MessageDoc

MessageDoc documents a protobuf message, its fields, and nested types.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Message name (e.g. "ForecastRequest"). |
| `description` | string | Message description from proto leading comments. |
| `fields` | repeated dotfilesd.v1.FieldDoc | Field documentation. |
| `nested_messages` | repeated dotfilesd.v1.MessageDoc | Nested message types. |
| `nested_enums` | repeated dotfilesd.v1.EnumDoc | Nested enum types. |

### FieldDoc

FieldDoc documents a single field in a protobuf message.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Field name (e.g. "location"). |
| `description` | string | Field description from proto leading comments. |
| `type` | string | Human-readable type string (e.g. "string", "int32", "weather.ForecastRequest"). |
| `label` | string | Label: "optional", "required", "repeated", or empty for proto3 default. |
| `is_oneof` | bool | If true, this field is part of a oneof group. |
| `oneof_name` | string | The oneof group name if is_oneof is true. |
| `default_value` | string | Default value if any (e.g. "true", "42"). |
| `message_doc` | dotfilesd.v1.MessageDoc |  |
| `enum_doc` | dotfilesd.v1.EnumDoc |  |
| `type_detail` | oneof | Full documentation of the field's type, populated recursively when the field is a message or enum type. |

### EnumDoc

EnumDoc documents a protobuf enum type.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Enum name (e.g. "Color"). |
| `description` | string | Enum description from proto leading comments. |
| `values` | repeated dotfilesd.v1.EnumValueDoc | Enum value documentation. |

### EnumValueDoc

EnumValueDoc documents a single enum value.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Value name (e.g. "COLOR_RED"). |
| `number` | int32 | Numeric value (e.g. 0). |
| `description` | string | Value description from proto leading comments. |
